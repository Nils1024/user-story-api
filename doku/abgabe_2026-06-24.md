## 2) Kernstatus in 6 Sätzen

1. **Verarbeitete Quellen:** CSV (Planner-Export-Simulation), JSON (GitHub-Issues-Export-Simulation) und XML (älteres Tool-Simulation). Alle drei Formate verwenden unterschiedliche Feldnamen und Strukturen.
2. **Stabiler Import:** Der `POST /import`-Endpunkt liest alle drei Formate zuverlässig ein – CSV mit UTF-8-BOM, JSON als Array von Objekten, XML über das `<user_story>`-Tag. Übersprungene Einträge ohne Titel werden korrekt gezählt.
3. **Gemeinsames Datenmodell:** Jedes Team erhält eine `UserStory` mit den Feldern `id` (int), `title` (string, Pflicht), `description` (string) und `classification` (Enum: SDM/GID/EVP). Die Klassifizierung erfolgt automatisch per Keyword-Regelwerk oder KI-API.
4. **Laufende API-Endpunkte:** GET `/userstories`, GET `/userstories/{id}`, GET `/userstories/fach/{fach}`, PATCH `/userstories/{id}/{fach}`, DELETE `/userstories/{id}` und POST `/import`. Alle testen sich gegen FastAPIs TestClient mit pytest.
5. **Fachzuordnung SDM/EvP/GiD:** Über das `_KEYWORDS`-Wörterbuch (60+ Keywords pro Fach) in Titel und Beschreibung. Jedes gefundene Keyword zählt +1, das Fach mit der höchsten Punktzahl gewinnt. Alternativ ruft die API ein externes LLM auf.
6. **Offenster Punkt:** Die KI-Klassifizierung ist konfigurationsabhängig – ohne gesetzte `AI_API_URL` und `AI_API_KEY` funktioniert sie nicht. Zudem fehlt noch ein Frontend zur interaktiven Verwaltung.

### Grenzfälle

**Grenzfall 1:** *„Als Manager möchte ich eine Benachrichtigung erhalten, wenn ein Bericht bereitsteht."*  
→ **Entscheidung: GID (primär)** aber mit EVP-Konflikt. Der Kontext („Bericht") dominiert als Geschäfts-dienst. Die Benachrichtigung ist hier sekundär – sie begleitet den Datenexport. In der Regel würden die Keywords „berichts" (+1) und „benachrichtigung" (+1) einen knappen Punktgewinn bedeuten, sodass das Fach mit dem breiteren Kontext gewinnt.

**Grenzfall 2:** *„Als Entwickler möchte ich ein Logging-System für alle API-Aufrufe implementieren."*  
→ **Entscheidung: SDM**. Das Keyword „logging" ist im SDM-Wörterbuch enthalten (Protokolle als Daten-management). Obwohl „API" im GID-Kontext stehen könnte, geht es hier um technische Systemarchitektur, nicht um einen Geschäftsdienst.
